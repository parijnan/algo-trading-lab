# Artemis Backtest — Iron Condor Historical Analysis

Part of the **Algo Trading Lab** project.

## Strategy Overview

A market-neutral Iron Condor on weekly options, deployed when India VIX < 16. Sells OTM options on both sides and buys further OTM options as hedges. Entry is fixed at Monday 10:30 AM; the position runs to Thursday expiry.

| Parameter | Value |
|---|---|
| Instruments | Nifty (Dec 2024 – Aug 2025) · Sensex (Sep 2025 – Mar 2026) |
| Deploy condition | India VIX < 16 |
| Structure | Iron Condor — PE spread + CE spread simultaneously |
| Strike selection | Scan OTM strikes at entry; pick closest LTP to `EXPECTED_PREMIUM` |
| Entry | Monday 10:30 AM signal → open of 10:31 candle |
| Expiry | Thursday 15:30 |
| VIX gate | Skip week if `vix_open >= VIX_THRESHOLD` on entry Monday |

## Stop Loss Mechanics

Both spreads are monitored independently on every 1-min candle. First condition to trigger exits that spread.

| Condition | Trigger | Guard |
|---|---|---|
| Index SL | Spot crosses within `INDEX_SL_OFFSET` pts of sell strike | Defer at 09:15 — re-check at 09:16, exit only if still breached |
| Option SL | `sell_ltp > sell_entry × sl_{dte}_dte` | Same 09:15 guard |

DTE-based option SL multipliers (weekday count, entry date → expiry date):

| DTE | Multiplier |
|---|---|
| ≥ 4 | `SL_4_DTE` (default 2.66) |
| 3 | `SL_3_DTE` (default 2.33) |
| 2 | `SL_2_DTE` (default 2.00) |
| 1 | `SL_1_DTE` (default 1.66) |
| 0 | `SL_0_DTE` (default 1.33) |

## Post-SL Behaviour

When one spread hits SL, the surviving spread is handled as follows:

| Surviving spread state | Action |
|---|---|
| Still active | `adjust_spread()` — roll sell strike; enter additional lots if before `cutoff_time` |
| Already closed | `reenter_spread()` — fresh strike selection and entry at current spot |

After `cutoff_time` (Wednesday 09:15): adjustments only, no additional lots.
After `elm_time` (Wednesday 15:15): no further adjustments.

## ELM Handling

At `elm_time`, the side with lesser net premium remaining (`sell_ltp − buy_ltp`) is fully exited. The surviving side has its hedge brought to within `HEDGE_POINTS` of the sell strike if needed. Additional lots on either side are also exited. Execution at open of the 15:16 candle.

## Execution Model

| Event | Execution |
|---|---|
| Entry | Open of 10:31 candle (signal at 10:30 close) |
| SL exit | Open of candle after SL detected |
| Adjustment / re-entry | Open of same candle as SL exit |
| ELM exit | Open of 15:16 candle |
| Expiry exit | Close of last available candle ≤ 15:30; `0.05` if missing |

## Optimisable Parameters

All live in `configs.py`. The analysis thread drives optimisation against these:

`EXPECTED_PREMIUM` · `HEDGE_POINTS` · `INDEX_SL_OFFSET` · `SL_0_DTE` through `SL_4_DTE` · `ADJUSTMENT_DISTANCE` · `MINIMUM_GAP` · `MINIMUM_GAP_ITERATOR`

## Setup

```bash
# 1. Generate contract schedule (run once; re-run if source files change)
python artemis_backtest/generate_contracts.py

# 2. Review and correct artemis_backtest/contracts.csv if needed

# 3. Set INSTRUMENT and date range in configs.py

# 4. Run backtest
python artemis_backtest/backtest.py
```

Dependencies: `pandas`, `numpy` (same environment as the rest of the lab).

## Trade Summary Schema

One row per weekly contract in `data/trade_summary.csv`.

| Column | Description |
|---|---|
| `instrument` | `nifty` or `sensex` |
| `expiry` | Thursday 15:30 timestamp |
| `entry_time` | Actual execution timestamp (10:31 open) |
| `entry_spot` | Index spot at 10:30 close |
| `entry_vix` | VIX open on entry Monday |
| `lots` | Base lot count (`LOT_COUNT` from configs) |
| `pe_sell_strike` / `pe_buy_strike` | Initial PE spread strikes |
| `ce_sell_strike` / `ce_buy_strike` | Initial CE spread strikes |
| `pe_sell_entry` / `pe_buy_entry` | PE entry prices |
| `ce_sell_entry` / `ce_buy_entry` | CE entry prices |
| `pe_sell_exit` / `pe_buy_exit` | PE exit prices |
| `ce_sell_exit` / `ce_buy_exit` | CE exit prices |
| `pe_exit_reason` / `ce_exit_reason` | `expiry` · `index_sl` · `option_sl` · `elm` |
| `pe_pl_points` / `ce_pl_points` | Base lots P&L per unit |
| `pe_add_pl_points` / `ce_add_pl_points` | Additional lots P&L per unit |
| `total_pl_points` | Combined P&L normalised to per-base-lot |
| `total_pl_rupees` | `total_pl_points × LOT_SIZE × lots` |
| `week_outcome` | `traded` · `skipped_vix` · `skipped_no_data` · `skipped_no_strikes` · `skipped_entry_failed` |
